# Contributing

Thank you for contributing to `kubenv`.

## Go style

This project follows the [Uber Go Style Guide](https://github.com/uber-go/guide/blob/master/style.md) for Go code style and conventions.

When you contribute Go code, use that guide as the default reference for:

- naming
- package structure
- error handling
- comments
- tests

If the repository already has a local pattern that is narrower than the guide, follow the repository pattern so the codebase stays consistent.
